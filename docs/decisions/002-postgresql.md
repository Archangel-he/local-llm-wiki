# ADR-002：PostgreSQL 作为业务主库

- 状态：Accepted
- 日期：2026-07-23

## 背景

项目早期考虑 DuckDB。DuckDB 适合嵌入式分析和单进程工作负载；目标系统包含 FastAPI、多个 Worker、用户、权限、任务状态和并发 Wiki 更新。

## 决策

- PostgreSQL 作为用户、空间、任务、Wiki、关系、引用和审计的主库；
- Alembic 管理 Schema；
- DuckDB 不进入在线请求路径，可在未来用于离线评测和 Parquet/CSV 分析。

## 后果

优点：

- 多进程客户端和 MVCC；
- 成熟事务、锁、约束、全文检索和权限能力；
- 与阿里云部署和未来托管数据库兼容。

代价：

- 本地多运行一个服务；
- 需要迁移、备份和连接管理。

Docker Compose 将本地运维成本控制为一条启动命令，因此代价可接受。

## 备选

- DuckDB：单机分析优秀，但不匹配目标在线事务负载；
- SQLite：原型简单，但未来并发和运维迁移成本较高；
- 文档/图数据库：当前需求没有证明需要。
